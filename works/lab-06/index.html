Preguntas

Respuestas a las preguntas de la actividad.

  • Es buena práctica checar que los inputs sean válidos por diferentes razones. Primero, siempre es buena idea revisar la validéz de los inputs. Segundo, conviene más que los datos sean validados por la parte del cliente en vez de que el servidor tenga que hacer más trabajo. Tercero, JS es un lenguaje no (débilmente) tipado, por lo que no se pueden tener varios supuestos que se tienen con lenguajes (fuertemente) tipados. Cuarto, es más rápido ya que se reduce la necesidad de comunicarse varias veces entre el servidor.
  • Hay varias formas de desactivar las validaciones. Primero, se puede desactivar el javascript. También se puede utilizar las herramientas de desarrollador para modificar el html y eliminar las validaciones. E incluso se pueden utilizar herramientas como postman o curl para enviar directamente las peticiones de http.
  • Las validaciones de javascript del lado del cliente no son la capa de seguridad principal, son solo una capa de conveniencia para facilitarle el trabajo al servidor. El servidor, independientemente de esto, debería de asumir que todo dato es potencialmete malicioso, por lo que debe validar todo.
